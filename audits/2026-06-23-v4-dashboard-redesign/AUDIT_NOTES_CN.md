# WeatherBot v4 看板审计记录

## 审计范围

- 页面：本地看板 `http://127.0.0.1:5173/`
- 流程：打开看板、判断当前是否可交易、查看天气趋势、查看信号、模拟买入、查看模拟记录、进入温度拟合分析。
- 本次证据：代码结构、类型构建、后端语法检查、单元测试。
- 限制：当前会话没有可用的 in-app browser 控制工具；后端重启命令被审批额度拦截，因此本轮未生成新的浏览器截图。

## 主要问题与处理

1. 首页信息架构过于拥挤。
   - 已处理：首页重排为左侧状态/模拟/实盘准备度，中间地球/天气趋势/资金曲线，右侧信号队列/交易记录。

2. 模拟与实盘边界不清。
   - 已处理：顶部和实盘准备度明确显示“当前只允许模拟观察”；信号里的实盘按钮保留但按门槛锁定。

3. 一键模拟状态不清。
   - 已处理：模拟账户说明改成“手动模式”，并展示最近一次一键模拟买入数、跳过数、原因和剩余现金。

4. 温度拟合页可能空白。
   - 已处理：后端不再只返回高置信校准样本，而是展示所有本地观察样本，并标记哪些不能用于实盘校准。

5. 天气模块没有展示机场/站点趋势。
   - 已处理：新增 `weather_city_series`，前端展示 Best/ECMWF/HRRR/METAR 曲线、站点、更新时间和湿度采集状态。

6. 信号表像风险标签堆叠。
   - 已处理：重写为行动队列，每行默认展示状态、价格、EV 和建议金额；展开后显示 Polymarket 链接、盘口、truth、分布和拦截原因。

## 可访问性注意点

- 图表旁增加了文字摘要和表格数据路径，避免只依赖 hover。
- 颜色状态配合文字标签，例如“建议先模拟”“实盘锁定”“仅观察”。
- 仍需浏览器验证：键盘焦点顺序、按钮可见焦点、窄屏 reflow、图表 tooltip 对键盘用户的替代路径。
