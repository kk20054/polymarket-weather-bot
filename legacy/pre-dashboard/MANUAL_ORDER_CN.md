# 半自动手动下单说明

当前版本不会自动提交真实订单。它发现买入信号时，会继续记录一笔本地模拟仓位，同时打印一张手动下单卡片。

## 信号里会显示什么

每个 `[BUY]` 后面会显示：

- Polymarket 事件页面 URL
- 具体市场问题
- 操作方向：BUY YES
- 建议限价
- 最大下单金额
- 预计 YES 份额
- 当前 bid/ask 和价差
- 天气桶、预测温度、EV、数据源
- YES token id，方便核对 CLOB 市场

## 如何手动下单

1. 打开 `URL`。
2. 找到信号里显示的同一个温度区间市场。
3. 选择 `Buy Yes`。
4. 使用 `Limit` 作为最高可接受价格。
5. 使用 `Amount` 作为最大投入金额。
6. 提交前再核对城市、日期、温度区间、价格和金额。

## 配置开关

在 `config.json` 里可以控制是否打印手动下单卡片：

```json
"manual_order_instructions": true
```

改成 `false` 后，只会显示原来的简短 `[BUY]` 行。

## 风险提醒

网页手动下单是真实交易，会使用你的 Polymarket 钱包资金。建议先用很小金额验证市场、价格和结算逻辑，再考虑扩大金额。
