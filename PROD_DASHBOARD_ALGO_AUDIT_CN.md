# WeatherBot 生产级看板与算法审计

日期：2026-06-21

## 当前结论

当前系统已经能作为本地模拟和人工观察工具使用，但还没有达到“无人值守自动赚钱”的生产级标准。主要原因不是界面不够漂亮，而是证据链还不够闭合：

- 看板能显示信号、模拟仓位、盈亏、EV、复盘和日志，但第一眼还不能完整回答“今天能不能自动下单”。
- 复盘样本仍偏少，当前已带实际温度的市场约十几个量级，不能据此放大实盘。
- 算法已经有盘口过滤、最低价过滤、模拟盘与实盘接口雏形，但缺少按城市、数据源、窗口、盘口质量分层后的自动降权。
- 天气市场存在传感器/观测站异常、临近结算跳变、盘口极薄、超低价 EV 失真等风险，必须进入交易前过滤。

## 一眼看懂程度

已经改善：

- 顶部固定显示扫描状态、数据新鲜度、模拟权益、已结算盈亏、胜率和持仓数。
- 模拟收益口径已从“现金 + 占用本金”改为“起始本金 + 已结算盈亏”，避免虚假盈利。
- 模拟记录可展开，能看到 Polymarket 链接、入场价、退出价、份额、数据源和结算原因。
- BTC 已从天气看板移除，避免策略混杂。
- 历史复盘模块增加“城市温度拟合”入口，用模型误差而不是只看 EV 评估策略。

仍不够生产级：

- 顶部缺少明确的“自动交易许可灯”：实盘关闭、AI 审核状态、风控熔断、今日额度、最小订单约束是否满足，应一屏可见。
- 信号表还需要把“为什么不买”与“为什么可以买”并列显示，例如 spread 过大、价格过低、样本不足、城市 MAE 过高。
- 日志区仍偏技术流，需要把错误聚合成可行动状态，例如“天气源失败”“盘口过期”“CLOB 余额不足”。
- 回测仍是本地模拟仓位复盘，不是逐分钟盘口回放，必须在 UI 中持续标注。

## 温度拟合页的作用

新增拟合页使用本地 `forecast_snapshots` 与 `actual_temp` 计算：

- `MAE`: 平均绝对误差，越低越适合做窄区间温度盘。
- `Bias`: 系统性偏热或偏冷，可用于城市级校准。
- `RMSE`: 对大错更敏感，用于识别异常城市或异常数据源。
- 城市排行：找出最不稳定城市，作为自动交易黑名单/降权来源。
- 数据源排行：比较 `model_best`、ECMWF、HRRR、METAR 的误差。

生产规则建议：

- 城市样本少于 10 个已结算市场：只允许模拟，不允许自动实盘。
- 城市 MAE 大于 3°F：自动实盘降权或禁止。
- Bias 绝对值大于 2°F：概率计算前应做城市级偏差校正。
- 临近结算时 METAR 与模型分歧大于 3°F：拒绝自动买入。

## 算法迭代方向

### 来自 X 帖子的可落地策略

本轮阅读了用户提供的 5 条 `@pritipatelfgoo` X 帖子，能落地为以下工程方向：

- NEAR-LOCK 策略：D+0 最后 6-18 小时优先使用结算机场 METAR。交易条件应变成 `hours_left < 18`、`observed_high + remaining_potential` 落入目标桶、`prob_bucket > 0.92`、`market_price < 0.82`。这不是普通全天候策略，应独立成 `near_lock` 信号类型。
- 集合离散度不足：24-48 小时窗口中，若集合预报 spread 明显小于该城市历史真实波动，则中间桶可能被高估、尾部桶可能被低估。工程指标为 `dispersion_ratio = historical_typical_spread / ensemble_spread`，当 `dispersion_ratio > 1.6` 且尾部价格低于 12-14c 时进入观察/模拟。
- 简单偏差校正优先：先做指数平滑 Bias、MOS 线性校正、卡尔曼滤波，不应一开始就上 ConvLSTM。当前新增的温度拟合页就是这些方法的基线数据面。
- 高级模型只作为后续：ConvLSTM + 自注意力 + 动态归一化适合样本量和数据管线成熟后再做，短期不应成为系统强依赖。
- 链上/排行榜观察：顶级交易员行为可以作为研究输入，但不能盲目跟单。后续可加“高手入场时间/仓位反应”观察面板，用于人工研究，不直接触发自动下单。

对应开发顺序：

1. 当前已完成：城市温度拟合页，显示 MAE、Bias、RMSE、城市排行、数据源排行。
2. 当前已完成：每条天气信号附带城市拟合质量：样本数、MAE、Bias、质量旗标。
3. 当前已完成：每条天气信号附带策略诊断：`near_lock_watch`、`near_lock_strong`、`dispersion_underpricing_watch`、`cheap_tail_candidate`、拟合风险等。
4. 当前已完成：温度拟合页增加策略回测切片，展示 NEAR-LOCK/METAR 误差和 D+1/D+2 离散度不足比例。
5. 下一步：把 `fit_sample_low / city_mae_high / city_bias_high / near_lock_missing_metar` 接入实盘风控；模拟盘继续允许记录，用来积累样本。
6. 下一步：新增独立 `near_lock` 扫描模块，用 METAR 当前最高温、剩余升温潜力、TAF 云量/风速过滤 D+0 临近结算市场。
7. 下一步：新增独立 `dispersion_underpricing` 模块，比较集合 spread 与城市历史 spread，专门寻找尾部错价。

### 1. 概率模型从单点预测升级为校准分布

当前风险：单点温度加固定 sigma 容易给出过高 EV。

建议：

- 每个城市维护 `city_bias_f` 和 `city_mae_f`。
- sigma 取 `max(ensemble_std, city_mae_f, source_mae_f, 1.5°F)`。
- 使用历史 Brier score 调整置信度。
- 对窄桶市场增加额外不确定性惩罚。

### 2. 交易信号从 EV 改为质量评分

建议评分：

`score = probability_edge * liquidity_score * calibration_score * freshness_score * source_agreement_score * microstructure_score`

禁止条件：

- ask < min_price
- ask > max_price
- spread > max_slippage
- orderbook 过期
- orderMinSize 不满足
- tick size 不匹配
- 城市拟合样本不足且未通过 AI 审核
- 传感器异常或临近结算数据源冲突

### 3. 低价极端盘单独处理

本轮亏损暴露出超低价 YES 盘会制造虚高 EV 和巨大份额。已经加入 `min_price=0.03`，后续还应增加：

- `max_shares_per_market`
- `min_best_bid`
- `min_exit_liquidity`
- 低价盘必须有盘口深度，不允许只看 ask。

### 4. AI 审核不直接替代量化规则

AI 第一阶段应只做审核和摘要：

- 汇总天气源冲突、城市误差、盘口异常、相关新闻/观测站异常。
- 输出结构化 JSON：批准、拒绝、置信度、否决理由。
- 默认只影响实盘，模拟盘仍记录以便 AB 对比。

## 下一步开发验收

短期必须完成：

- 在信号表显示质量评分和拒绝原因。
- 将温度拟合指标接入信号生成，城市 MAE/Bias 影响概率。
- 看板顶部新增“自动交易许可灯”。
- 每日飞书摘要包含拟合误差、拒绝原因分布、模拟/实盘表现。

实盘前验收：

- 至少 7 天稳定模拟。
- 至少 30 个已结算模拟仓位。
- 城市拟合页样本和误差稳定可解释。
- dry-run 下单、重复订单保护、orderMinSize/tick size 校验通过。
- 第一笔实盘只允许 1-2 美元 canary。
